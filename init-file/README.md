# Software Engineer Interview Prep System

এটা একটা **রিইউজেবল, লং-টার্ম ইন্টারভিউ প্রস্তুতি সিস্টেম**। এখানে এখনই সব কনটেন্ট জেনারেট করা হচ্ছে না — এটা শুধু একটা স্ট্রাকচার/স্পেক, যা VS Code-এ Claude Code দিয়ে খুলে প্রয়োজন অনুযায়ী প্রতিটা টপিকের কনটেন্ট জেনারেট করার জন্য ব্যবহার করা হবে।

## কীভাবে ব্যবহার করবে
1. এই ফোল্ডারটা VS Code-এ ওপেন করো (Claude Code এক্সটেনশন/CLI দিয়ে)
2. `topics/` ফোল্ডার থেকে যে টপিকটা এখন পড়তে চাও, সেই ফাইলটা ওপেন করে Claude Code-কে বলো:
   *"এই README.md-এর Answer Pattern অনুযায়ী এই ফাইলে কনটেন্ট জেনারেট করো"*
3. একবারে একটা টপিক জেনারেট করাই ভালো — ইন্টারভিউর আগে সময় বাঁচানোর জন্য
4. জেনারেট হয়ে গেলে `TODO.md`-এ সেই টপিকটা টিক দিয়ে রাখো

## Answer Pattern (কনটেন্ট জেনারেট করার নিয়ম)

### সাধারণ টপিকের জন্য (Laravel, React, Node.js, Python, DB, AWS, Kafka/Redis/RabbitMQ)
প্রতিটা টপিকে ১০টা প্রশ্ন, প্রতিটা প্রশ্নের ফরম্যাট:

```
### [নম্বর]. [প্রশ্নের টাইটেল]

**Description:** সহজ ভাষায় ২-৩ লাইনে প্রশ্নটা কী জিজ্ঞেস করছে এবং কেন গুরুত্বপূর্ণ, সেটার ব্যাখ্যা

**৬টা Key Point:**
1. ...
2. ...
3. ...
4. ...
5. ...
6. ...
```

### জটিল টপিকের জন্য (System Design, Design Pattern)
পূর্ণ ব্যাখ্যার বদলে **description + মুখস্থ করার পয়েন্ট** ফরম্যাটে:

```
### [নম্বর]. [প্রশ্নের/সিনারিওর টাইটেল]

**Description:** সংক্ষেপে সমস্যাটা কী এবং কী ধরনের সমাধান আশা করা হচ্ছে

**মনে রাখার পয়েন্ট:**
- ...
- ...
```

### Problem Solving টপিকের জন্য (আলাদা ফরম্যাট)
টপ ৫টা প্রবলেম, প্রতিটায়:
```
### [নম্বর]. [প্রবলেমের নাম]

**Description:** প্রবলেম স্টেটমেন্ট সহজ ভাষায়

**কোড:**
```language
// সমাধান কোড
```

**ব্যাখ্যা:** কোডটা কীভাবে কাজ করছে, Time/Space Complexity
```

### System Design (বাংলাদেশ মার্কেট কনটেক্সট) — টপ ৫টা সিনারিও
নির্দিষ্টভাবে এই ৫টা এরিয়া কভার করবে:
1. Order & Stock Management System
2. Ticket Management System
3. Heavy-load Background Job — Report Generation
4. User CSV Registration (with Notification)
5. Failed Job Handling ও Backup Plan

প্রতিটায় description + মনে রাখার পয়েন্ট ফরম্যাট ব্যবহার হবে।

### HR প্রশ্নের জন্য
প্রশ্ন + উত্তরের কাঠামো (framework), ব্যক্তিগত প্রেক্ষাপট (বর্তমান রোল, বেতন প্রত্যাশা, নিজের কোম্পানি ইত্যাদি) বিবেচনা করে।

## সাধারণ নিয়ম (সব টপিকে প্রযোজ্য)
- **সব কনটেন্ট বাংলায়** (টেকনিক্যাল টার্ম ইংরেজিতে থাকবে, যেমন Middleware, Index)
- **অডিয়েন্স:** সিনিয়র ইঞ্জিনিয়ার — তাই বেসিক জিনিস ব্যাখ্যা না করে সরাসরি গভীর/practical পয়েন্টে যাওয়া
- **সহজ ও বোধগম্য** ভাষা — জটিল বাক্য এড়িয়ে চলা
- প্রতিটা টপিকে সময় বরাদ্দ: **১৫ মিনিট** (রিভিশনের জন্য প্র্যাকটিক্যাল সময়সীমা)
- সম্ভব হলে প্রতিটা পয়েন্টে বাস্তব প্রজেক্ট এক্সপেরিয়েন্স থেকে ছোট উদাহরণ যোগ করা যেতে পারে
